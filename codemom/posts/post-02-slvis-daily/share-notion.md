# `/daily` 스킬 내 입맛에 맞게 세팅하는 법

> 인스타에서 본 그 `/daily` 스킬을 내 취향대로 세팅하는 가이드예요.
> 같은 스킬이라도 **내가 뭘 적어두느냐에 따라** 매일 아침 받는 브리핑이 달라져요.

---

## 핵심 아이디어

이 스킬은 두 개의 파일로 굴러가요.

1. **내가 누구인지 적은 파일** (`CLAUDE.md` + `사용자 프로필.md`)
2. **스킬이 뭘 할지 적은 파일** (`SKILL.md`)

AI는 `/daily`를 칠 때마다 **1번 파일을 먼저 읽고** → **2번 파일의 절차를 따라** 브리핑을 만들어요.

그래서 **1번 파일을 내 취향대로 채우는 게 전부**예요. 스킬 로직은 한 번 깔고 거의 안 건드려도 되고요.

---

## 준비물

- **Claude Code** — CLI 설치 (`claude.ai/code`)
- **Obsidian** + `obsidian-cli` — 내 프로필·일일 노트 저장용
- **Google Calendar MCP** — `/mcp` 명령으로 연결
- **웹 검색 권한** — `WebFetch` / `WebSearch`

> Obsidian을 쓰지 않으면 프로필을 `CLAUDE.md` 안에 직접 적어도 돼요. 이 문서는 Obsidian 기준으로 설명해요.

---

## 폴더 구조

```
~/my-ai/                       ← 프로젝트 루트 (이름 자유)
  CLAUDE.md                    ← 내 비서 설정
  .claude/skills/daily/
    SKILL.md                   ← /daily 스킬 본체
```

그리고 Obsidian vault 안에는

```
<비서_이름>/
  사용자 프로필.md             ← 내 취향의 뿌리
  운영 가이드.md               ← 비서 행동 규칙
```

---

## STEP 1 · `CLAUDE.md` — 내 비서 기본 설정

`~/my-ai/CLAUDE.md`에 저장하세요. `<>` 부분만 본인 걸로 교체.

````md
# <비서\_이름>

<한 줄 컨셉 — 예: "아이언맨 자비스에서 영감받은 내 전용 비서">
한국어 존댓말로 대화하며, 사용자를 항상 **<내 호칭>**으로 부른다.

## 핵심 역할

1. 일상 관리 — 일정, 할 일, 일일 브리핑, 뉴스 큐레이션
2. 지식 관리 — Obsidian vault 기반 기록·검색·연결
3. <내가 원하는 역할 추가 — 블로그, 투자, 육아 등>

## 대화 시작 시 필수 조회 (매번)

```bash
obsidian read path="<비서_이름>/사용자 프로필.md"
obsidian read path="<비서_이름>/운영 가이드.md"
```

## 도구

### Obsidian CLI

```bash
obsidian read path="폴더/파일.md"
obsidian append path="폴더/파일.md" content="추가 내용"
obsidian daily:append content="내용"
obsidian search query="검색어"
```

### Google Calendar

내 캘린더: `<내-gmail>` (개인) · `<공유캘린더ID>` (가족 등) · `ko.south_korea#holiday@group.v.calendar.google.com` (공휴일)

## 행동 원칙

1. 간결함 — 핵심만, 장황하지 않게
2. 기록 우선 — 중요 정보는 바로 노트에
3. 태그 일관성 — `#<내가_쓰는_태그>`
````

## STEP 2 · `사용자 프로필.md` — **내 입맛을 결정하는 파일**

이게 제일 중요해요. 여기에 뭘 적느냐에 따라 매일 아침 브리핑이 달라져요.

Obsidian vault의 `<비서_이름>/사용자 프로필.md`에 저장하세요.

```md
## 나에 대해

- 이름 / 호칭
- 직업 · 현재 상태
- 관심 분야 (3~5개)
- 집중하고 있는 프로젝트

## 뉴스 큐레이션 선호

### AI/기술

- 관심: <예: AI 에이전트, LLM, 개발자 도구, 프론트엔드>
- 제외: <예: 암호화폐, 하드웨어 가십>
- 소스: <예: news.hada.io, HackerNews>

### 경제

- 관심: <예: 거시지표, 금리, 환율, 반도체>
- 스타일: <예: 해설보다 팩트 위주, 그래프 언급 환영>

### 정치

- 관심: <예: 정책 변화, 국제 관계>
- 스타일: <예: 특정 매체 편향 피하기, 좌·우 섞기>

## 톤 선호

- 존댓말 캐주얼 / 격식
- 이모지: 0개 / 적절히
- 문장 길이: 짧게 / 자세하게
- 싫어하는 것: <예: 장황한 설명, "오늘도 화이팅" 같은 격려 문구>
```

`/daily`를 칠 때마다 AI가 이 파일을 먼저 읽기 때문에, 파일만 업데이트해도 내일 아침 브리핑이 다르게 나와요.

## STEP 3 · `SKILL.md` — 스킬 본체

`~/my-ai/.claude/skills/daily/SKILL.md`에 저장. 이 부분은 **거의 그대로 써도 돼요.** 캘린더 ID랑 뉴스 소스만 본인 걸로 바꾸면 끝.

```md
---
name: daily
description: 오늘의 일일 브리핑을 수행한다. "/daily" 또는 "일일 브리핑", "오늘 일정", "뉴스 큐레이션" 요청 시 사용. 선택 인수로 날짜(YYYY-MM-DD) 지정 가능, 생략 시 오늘.
allowed-tools:
  - Bash(obsidian *)
  - mcp__claude_ai_Google_Calendar__list_events
  - mcp__claude_ai_Google_Calendar__list_calendars
  - WebFetch
  - WebSearch
---

오늘의 일일 브리핑을 수행한다. 분석하지 말고 정해진 절차만 따른다.

대상 날짜: `$ARGUMENTS` (없으면 오늘)

## 1단계: 일일 노트 확인

1. `obsidian daily:read`로 오늘 일일 노트를 읽는다
2. 없으면 템플릿으로 생성
3. 이미 브리핑 내용이 채워져 있으면 사용자에게 알리고 중단

## 2단계: Google 캘린더 조회

Google Calendar MCP로 병렬 조회:

- 개인: `calendarId` 생략 (기본 = primary)
- 공유: `calendarId = "<공유캘린더ID>"`
- 공휴일: `calendarId = "ko.south_korea#holiday@group.v.calendar.google.com"`

공통 파라미터:

- `startTime`: 대상날짜 `T00:00:00`
- `endTime`: 대상날짜+1일 `T00:00:00`
- `timeZone`: `Asia/Seoul`
- `orderBy`: `startTime`

일정 없으면 "일정 없음"으로 표시.

## 3단계: 뉴스 큐레이션

### AI/기술

1. `WebFetch`로 <내가 정한 기술 뉴스 URL> 조회
2. **사용자 프로필의 관심사 기준**으로 3~5개 선별
3. 형식: `- [제목](URL) — 한 줄 요약`

### 경제/정치

1. `WebSearch`로 오늘의 주요 뉴스 검색
2. **사용자 프로필의 스타일 선호 반영**해서 3~5개 선별
3. 형식: `- [제목](출처URL) — 한 줄 요약`

## 4단계: 일일 노트 저장

`obsidian daily:append`로 저장.

## 5단계: 브리핑 출력

저장한 내용을 그대로 출력. 추가 분석·코멘트 불필요.
```

## 입맛에 맞게 바꾸는 포인트

### 1. 뉴스 소스

`WebFetch` URL을 내가 자주 보는 사이트로 바꾸세요.

- AI·기술 → `news.hada.io`, `news.ycombinator.com`, `techcrunch.com`
- 경제 → 한경·매경 실시간, `investing.com`
- 디자인 → `designmodo.com`, `sidebar.io`

여러 개 쓰고 싶으면 3단계에 소스를 나열하고 "각 소스에서 1~2개씩" 같은 규칙을 추가해요.

### 2. 카테고리

경제·정치가 꼭 있어야 하는 건 아니에요. 본인 관심사로 교체:

- 부동산 시세
- 스포츠 (특정 팀만)
- K-콘텐츠 (드라마·OTT 순위)
- 학계 논문 (arXiv 특정 카테고리)

3단계의 섹션 이름과 프롬프트만 바꾸면 돼요.

### 3. 요약 형식

- `한 줄 요약` → `두 줄 요약` 또는 `불릿 3개`
- 중요도 표시 추가: `[중요] 제목` 형식으로
- 읽기 예상 시간: `(2분)` 같은 태그

### 4. 추가 단계

1단계 앞/뒤에 새 단계를 끼워 넣을 수 있어요.

- **0단계**: 어제 일일 노트에서 미완료 할 일 가져오기
- **3.5단계**: 날씨 조회 (`WebFetch` 기상청)
- **6단계**: 오늘의 한 마디 (사용자 프로필의 목표 기반 리마인더)

### 5. `사용자 프로필.md`만 고쳐도 되는 것들

아래는 SKILL.md는 그대로 두고 프로필만 바꿔도 반영돼요.

- 뉴스 선별 기준 (관심·제외 키워드)
- 톤 (격식/캐주얼)
- 특정 회사·인물 감시 리스트
- 제외할 주제 (예: "부고 뉴스 제외")

## 자주 묻는 질문

**Q. `/daily`가 안 떠요**
→ `.claude/skills/daily/SKILL.md` 경로가 맞는지, frontmatter의 `name: daily`가 맞는지 확인해주세요. `claude` 재실행도.

**Q. Obsidian 없이도 돼요?**
→ 돼요. `SKILL.md`에서 1단계·4단계(노트 저장) 제거하고, 사용자 프로필은 `CLAUDE.md` 안에 직접 적으세요.

**Q. 캘린더 ID는 어디서 찾아요?**
→ Claude Code에서 `mcp__claude_ai_Google_Calendar__list_calendars`를 한 번 호출하면 전체 목록과 ID가 나와요.

**Q. 뉴스 링크 대신 요약 본문만 붙게 하려면?**
→ 3단계 출력 형식을 `- 제목 — 요약 본문 (2~3줄)`로 바꾸고, "링크는 문서 하단에 모아서"라는 규칙을 추가하면 돼요.

**Q. 매일 같은 뉴스만 떠요**
→ 사용자 프로필에 "어제 브리핑과 **다른 기사 우선**" 규칙을 추가하거나, SKILL.md 1단계에서 어제 일일 노트도 읽도록 해서 중복 체크를 시키세요.

## 정리

- **스킬 로직**은 한 번 깔고 끝
- **취향**은 `사용자 프로필.md`에 계속 쌓기
- AI를 학습시키는 게 아니라 **파일에 적어서 매번 읽히는 방식**이라, 이동·복제·공유가 쉬워요
