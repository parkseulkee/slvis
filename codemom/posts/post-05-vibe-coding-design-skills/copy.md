## 4. 예상 질문 대응

### Q. 초보인데 쓸 수 있나요?

```
claude code 써본 적 있으면 충분해요.
CLAUDE.md에 한 줄만 넣으면 스킬이 자동으로 호출되니까,
별도 학습 없이 바로 쓰실 수 있어요.
```

### Q. Cursor 에서도 되나요?

```
이 두 스킬은 Claude Code 기반이에요.
Cursor에선 .cursorrules 로 비슷한 규칙을 만들 수는 있지만,
동일 재현은 Claude Code 에서 가장 정확해요.
```

### Q. 프로젝트마다 다르게 쓰고 싶어요

```
스킬 자체는 ~/.claude/skills/ 에 한 번 설치하고,
프로젝트별로 CLAUDE.md 한 줄만 조정하면 돼요.

호출 규칙을 프로젝트마다 다르게 쓰실 수 있어요.
```

### Q. 설치가 잘 안 돼요

```
어떤 메시지가 뜨는지 스크린샷 한 장 보내주실래요?
~/.claude/skills/ 경로에 폴더가 잘 들어갔는지,
그리고 프로젝트 CLAUDE.md 가 인식되는지 순서대로 보면 보통 해결돼요.
```

### Q. 스킬 내용을 직접 고쳐도 되나요?

```
네, SKILL.md 를 프로젝트 톤에 맞게 수정해서 쓰셔도 돼요.
저도 프로젝트마다 조금씩 다르게 쓰고 있어요.
```

## 5. 첨부 파일 준비 체크리스트

링크 / 이메일 방식 상관없이 아래가 준비돼 있어야:

- [ ] `brainstorming/` 폴더 전체 — `SKILL.md` + `references/` + `scripts/`
- [ ] `explore-plan-code/` 폴더 전체 — `SKILL.md`
- [ ] `CLAUDE.md-template.md` — 위 메시지 3의 한 줄 예시
- [ ] `README.md` — 3~4줄 설치 가이드 (위 메시지 2 · 3 내용 요약)

### zip 압축 (로컬에서)

```bash
cd ~/.claude/skills
zip -r brainstorming.zip brainstorming
zip -r explore-plan-code.zip explore-plan-code
```

두 zip + CLAUDE.md-template.md + README.md 를 Drive/Gist/Notion 한 곳에 묶어두기.

## 6. 톤 체크

DM 카피 전부 아래 룰 준수:

- 존댓말 캐주얼 (`~했어요 / ~드려요 / ~주세요`)
- 이모지 0
- `ㅋㅋ / ㄷㄷ / ㅠㅠ` 0
- 느낌표 최소 (필요 시 메시지당 0~1번)
- 호들갑·과장 없음

독자가 "광고체 답변이네" 라고 느끼지 않게, 평범한 개발자 선배의 담담한 안내 톤 유지.
