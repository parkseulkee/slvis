## Phase 2 — 디자인 시스템 구축

### 핵심 결정 — Dual-Theme

하나의 타이포/그리드/로고를 공유하면서, 컨텐츠 축에 따라 팔레트와 모드가 오가는 구조:

| Mode | 배경 | 용도 |
|---|---|---|
| LIFE | 크림 `#FAF7F2` | 일상, 육아 |
| TECH | 딥블랙 `#0E0D0A` | AI, 개발 |

**공유 뉴트럴**: `#2E2A22` (웜블랙) — 두 모드에서 모두 텍스트/스트로크로. 이게 있어서 계정이 하나로 묶여 보임.

### 만든 것

- `colors_and_type.css` — 토큰 (색, 타입 스케일 112 → 16, 간격, 반경, 그림자)
- `assets/` — 로고 3종 + 디바이더
- `preview/` — 디자인 시스템 프리뷰 18장 (Type / Color / Spacing / Components / Brand)
- `slides/` — 1080×1350 템플릿 **12종**
- `ui_kits/instagram/` — 피드 모형

### 반복 피드백

첫 시안에서 Claude가 "감성적인" 세리프(Fraunces, 이태릭)를 적극 넣었는데, 이때 피드백:

> D 가 좋아. 너무 감성적인건 일상에도 싫어.

→ **Space Grotesk Bold (히어로) + Pretendard (본문) + JetBrains Mono (라벨)** 로 통일. 세리프/이태릭/손글씨 전부 제거.

이후 피드백들:

- 배경 카드에 용도 라벨이 없다고 지적 → 각 배경마다 "언제 쓰는지" 라벨 추가
- 커버 슬라이드의 사진 위치 상단으로 이동 요청 → 사진 상단, 타이포 하단 구성으로 정착
- 커버 타이틀 줄바꿈이 어색하게 잘린다는 지적 → h1 스케일 축소, 3줄 압축
- 사진을 반투명 배경으로 깔고 텍스트를 오버레이하는 방향 제안 → 풀블리드 + 다크 그라디언트 오버레이로 재작업

## Phase 3 — 실제 포스트 제작 (Post 01)

### 프롬프트

> 이 포스팅 기획을 따라서 만들어줘. (기획서 첨부)

기획서에는 별도 팔레트(라벤더/민트 등)가 있었는데:

> 아 기획서에 있는 색상은 무시하고, 우리가 정한 컬러로 해.

→ 시리즈 일관성이 훨씬 중요. Dual-Theme 팔레트로 재매핑.

### 중요한 톤 피드백들

**"ㅋㅋ" 문제:**

> 생각해보니 'ㅋㅋ' 이게 지금 디자인이랑 안맞는거 같아. 전체 슬라이드 디자인 톤에 텍스트 톤을 맞춰줘

→ `ㅋㅋ` 삭제, `… 저요.` 로 교체. 한 박자 뒤 담담한 인정.

**자기선언 라벨 제거:**

> 사용자가 늘라기를 바라는 지점을 레이블이나 문구로 직접 선언하지 말고, 구조와 카피로 느껴지게 해주세요. 이모지와 감정 유도 도형도 제거해주세요.

→ `PLOT TWIST` 배지, ⚡, 😊, 기울어진 말풍선 모두 제거. 대신 `— 그런데` 에디토리얼 캡션 + 큰 문장 하나.

**고유명사 처리:**

> 고유명사는 독음을 괄호로 병기하지 말고, 그 단어가 담고 있는 의미를 카피로 풀어서 보여주세요.

→ `JANJAN (잔잔)` 독음 제거. `A diary for the quiet days.` 태그라인 + `~~흘려보내던~~ 잔잔한 하루를 **기록**하게` 로 의미를 카피로 표현.

### 이 과정에서 `TONE_GUIDE.md` 탄생

피드백이 반복되다 보니 금기/선호 규칙이 쌓였고, 이걸 별도 문서로 정리:

- ❌ ㅋㅋ / ㄷㄷ / 이모지 / 자기선언 라벨 / 고유명사 독음 병기 / 감정유도 도형
- ✅ 담담한 자조 / 몽타주식 문장 / 에디토리얼 캡션 / 의미를 카피로

## 배운 것 3가지

### 1. "Decide for me"는 위험한 동시에 좋다

초반에 "팔레트는 Decide for me"라고 맡겼을 때 Claude가 과감하게 **Dual-Theme**이라는 구조를 제안했음. 이게 아니었으면 한 톤만 선택해야 했을 것. 다만 감성 톤이 과했고 거기서부터 피드백 싸이클이 시작됨.

→ 교훈: **방향은 AI에 맡기되, 결과물에 반드시 돌려서 반복 피드백.**

### 2. 톤은 한 번에 안 잡힌다

`ㅋㅋ`, `PLOT TWIST`, `JANJAN (잔잔)`, 이모지, 감정 유도 도형 — 전부 Claude가 처음에 넣었다가 내가 하나씩 뺐음. 이 패턴을 `TONE_GUIDE.md`로 뽑아내니까 이후 포스트에서 같은 실수 안 나옴.

→ 교훈: **거부한 표현들을 규칙으로 명문화해두면, 다음 포스트의 초안부터 바로 맞는 톤이 나온다.**

### 3. 디자인 시스템 + 톤 가이드 + 스킬 패키지 = 확장 가능한 계정 운영

초반 며칠간 디자인 시스템을 제대로 만들고 나니, 이후 포스트는 **"이 이야기를 카드뉴스로"** 한 줄에서 시작해도 톤이 깨지지 않게 됨. 스킬화까지 하면 로컬 `claude code`로 무한 확장.

→ 교훈: **처음에 느리게 가면 나중에 빠르게 간다.**
