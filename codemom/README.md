# @codemom.life — Design System

인스타그램 카드뉴스(캐러셀) 를 위한 디자인 시스템입니다. 계정 운영자는 9년차 개발자이자 육아휴직 중인 개발자맘이며, 4개 축(개발자로서의 나 / AI 활용 / 일상 / 내가 만드는 앱들)의 컨텐츠를 1080×1350(4:5) 포맷으로 꾸준히 발행합니다.

## 핵심 아이디어 — Dual-Theme System

하나의 타이포/그리드/로고를 공유하면서, 컨텐츠 축에 따라 **두 개의 팔레트와 모드**를 오가는 시스템입니다.

| Mode | 쓰는 축 | 배경 | 분위기 |
|---|---|---|---|
| **LIFE** (warm) | 일상, 육아, 감성 | 크림(#FAF7F2) | 베이지/크림, 세리프 이태릭, 폴라로이드, 손글씨 느낌 |
| **TECH** (dark) | AI, 개발 팁, 툴 리뷰 | 딥블랙(#0E0D0A) | 모노스페이스 프롬프트, 터미널 도트, 앰버+그린 악센트 |

두 모드는 **cream(#F2ECE0)** 과 **warm-black(#2E2A22)** 을 공유 색으로 삼아 계정 전체가 하나의 브랜드로 보이게 묶입니다.

## CONTENT FUNDAMENTALS — 카피 원칙

**어조:** 존댓말 캐주얼. "~했어요 / ~해봤어요 / ~하더라고요" — 존댓말이지만 친구처럼 말 거는 톤. "~합니다" 정도의 격식은 쓰지 않습니다.

**인칭:** "저는"은 자주 생략. "새벽 2시에 일어나요" 처럼 주어 없이 자연스럽게. 독자에게는 "여러분"보다 **"~보세요 / ~해보세요"** 로 2인칭 암시.

**제목 공식 (훅 패턴):**
- 숫자 고정형 · "매일 쓰는 AI 도구 **5가지**"
- 대비형 · "예상 vs 현실", "대기업 **vs** 1인 개발"
- 시간 고정형 · "새벽 **2시**의 코딩 루틴"
- 질문 던지기 · "AI가 **아직** 못하는 건?"
- 역설형 · "9년차 개발자가 육아휴직 중에 하는 일"

**본문 스타일:**
- 한 슬라이드 한 생각. 두 문장 넘기지 않기.
- 숫자는 자주 쓰기 — "D+127", "새벽 2:00", "~10.5h", "1,247명"
- 코드/도구명은 **모노스페이스**로 — "`useEffect`", "`claude code`", "`pnpm dev`"
- 은유 사용 OK · "좋은 코드는 아이처럼 자주 봐줘야 해요"

**이모지:** **거의 안 씀.** 대신 **mono 기호**로 대체 — `→ · — · / · $ · # · ←` 등. 마지막 CTA 슬라이드에서만 아이콘(하트/북마크) 사용.

**해시태그:** 포스트당 5~8개. 핵심 태그 — `#코드맘 #개발자맘 #육아휴직 #개발자의하루 #AI활용 #코드맘이만든앱`

**금기:**
- "안녕하세요 구독자 여러분!" 같은 유튜브식 오프닝
- "오늘은 ~에 대해 알려드리겠습니다" 같은 격식 도입
- 느낌표 남발, 물결(~) 과다
- "정답은 없지만..." 같은 에두르기

## VISUAL FOUNDATIONS

### 색
- **LIFE 배경:** `#FAF7F2` (크림) → `#F2ECE0` (베이지 서피스) → `#E8DFCA` (베이지-2)
- **TECH 배경:** `#0E0D0A` (딥) → `#17150F` (서피스) → `#22201A` (서피스-2)
- **공유 뉴트럴:** `#2E2A22` (웜블랙 · 두 모드 모두에서 텍스트/스트로크)
- **LIFE 악센트:** `#C4461F` (테라코타) · 포인트 컬러 하나만
- **TECH 악센트:** `#E8B960` (앰버 터미널) + `#7FB069` (터미널 그린) 보조
- **강조 배경 전용:** `#F4E4C1` (버터 하이라이트), sunset gradient (표지 CTA 전용)
- **시스템:** `#6B8A3E` (OK/올리브), `#E07856` (경고/엠버)

### 타입
- **Pretendard Variable** — 본문, 일반 타이틀 (한글 기본)
- **Space Grotesk** — 히어로/디스플레이 타이틀 (영문 임팩트)
- **JetBrains Mono** — 라벨, 터미널 텍스트, 페이지 번호, 툴명

타입 스케일은 1080×1350 캔버스 기준: 112 → 84 → 64 → 52 → 40 → 34 → 30 → 24 → 20 → 16. 슬라이드에선 **24px 이하 금지**.

### 배경
- 이미지는 **4:5 또는 1:1 사진** — 사용자 직접 촬영. 플레이스홀더는 크로스해치 패턴 + 점선 프레임.
- **그래디언트는 표지/CTA에만** — 테라코타 sunset이 대표. 일반 슬라이드에서 남용 금지.
- **그리드 라인:** TECH 모드 배경에 60px 서브틀 그리드(opacity 0.04). LIFE는 그리드 없음.
- **그레인:** `.grain` 유틸로 3.5% 노이즈 오버레이. 인쇄감 부여용.

### 애니메이션/hover
- 정적 슬라이드. 애니메이션 없음. 예외: **TECH 표지의 `█` 블링크 커서**.
- 인터랙티브 프로토타입에선 `transform: scale(0.98)` + opacity 0.8의 미묘한 press.

### 테두리·그림자
- 카드 = `var(--sh-card)` (warm tone, 2단 그림자)
- 표지 큰 카드 = `var(--sh-lift)` 
- TECH 카드 = `var(--sh-tech)` (검은 배경 위에 앰버 링 1px)
- 테두리 = `#D9CFB8` (LIFE), `#22201A` (TECH) — **얇게 1px 또는 dashed**

### 반경
대부분 `14~24px`. 표지 대형 카드는 `36px`. 배지·버튼은 `pill`. 

### 레이아웃 룰
- 캔버스 **1080×1350**
- 안전 영역 패딩 **72px 좌우 / 96px 상하**
- 모든 슬라이드에 **상단 mono 라벨** (`DEV · D3` 같은) + **하단 @codemom.life + SWIPE →** 고정
- 페이지 카운터 형식 `3 / 7` 또는 `# 03 / 08`

### 투명도·블러
- 사진 위 텍스트 올릴 때 **크림색 캡슐(rgba(250,247,242,0.85))** — 블러 X, 그냥 솔리드.
- TECH 모드에서 `rgba(232,185,96,0.15)` 링으로 terminal-glow 힌트.

## ICONOGRAPHY

- **Lucide** 스타일 기반, **stroke-width 1.75, round-cap, round-join**. 직접 SVG 사용 (CDN 임포트 가능: `https://unpkg.com/lucide@latest`).
- **사용 양:** 최소. 대부분의 슬라이드는 아이콘 없이 타입+숫자만.
- **허용 용도:** CTA 슬라이드 (하트/북마크/화살표), 타임라인 시간 표시 (시계), 리스트 번호 옆 토큰.
- **이모지:** **쓰지 않음.** 이모지가 들어가면 브랜드 톤 깨짐.
- **유니코드 대체:** `→ ← ● ○ ★ ☆ — ·` 자주 사용. mono 폰트로 렌더해서 결 맞춤.

### 키 에셋

## 주의사항
- 폰트 TTF 파일은 포함하지 않고 **웹폰트(Pretendard CDN, Google Fonts)** 로 로드합니다. 피그마 사용 시 Pretendard Variable은 [공식 배포처](https://github.com/orioncactus/pretendard)에서 설치하세요.
- 사진은 전부 **플레이스홀더**로 처리되어 있습니다. 실제 포스팅 때 4:5 비율로 교체하세요.
- 슬라이드 번호 카운터(`3 / 7`)는 캐러셀 장수에 맞춰 수동으로 갱신해야 합니다.
